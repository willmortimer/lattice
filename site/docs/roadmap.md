---
title: Product direction
description: What Lattice is improving next across the local app, open ecosystem, and optional connected services.
---

Lattice already spans local notes, canvas, relational data, analytical data,
notebooks, tasks, workflows, proposals, interfaces, and artifacts. The next work
is less about adding isolated engines and more about making those engines compose
beautifully.

## Near-term focus

- Polish capture, writing, search, navigation, Inspect, and release packaging.
- Make tasks, workflows, proposals, artifacts, interfaces, and live embeds feel
  like one coherent programmable workspace.
- Add lineage and staleness so generated outputs explain their inputs and rebuild path.
- Improve cross-resource dashboards, bindings, actions, and review surfaces.
- Expand user docs, import quality, conformance fixtures, and performance evidence.

## Product horizon

- Static publishing and complete Lattice App packages.
- Plugin and capability-pack ecosystem.
- Optional personal sync and team collaboration.
- Mobile capture, reading, and native ink.
- Richer remote data, semantic models, and managed execution.

This is direction, not a date or availability promise. The canonical
[engineering roadmap](https://github.com/willmortimer/lattice/blob/main/docs/29-roadmap.md)
retains deeper architecture sequencing.
